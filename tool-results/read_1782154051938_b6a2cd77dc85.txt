  1022→4. Dynamic Regional Engine & Multi-Data-Plane Compliance
  1023→4.1 Overview
  1024→Circle operates in a world of conflicting national laws, censorship regimes, and payment systems. Instead of building separate apps or requiring users to choose a “region,” Circle includes a Dynamic Regional Engine (DRE) . The DRE fetches a JSON configuration file based on the user’s IPderived country (or manually selected preference) and instantly adapts every module – from chat encryption to payment methods to content filtering – without requiring an app update.
  1025→This design allows Circle to:
  1026→Comply with local laws (e.g., China’s realname requirements, EU’s GDPR)
  1027→Federate where permitted, or operate as a closed network where required
  1028→Show only relevant payment methods and cultural features
  1029→Block or modify features in sanctioned regions (e.g., Iran) without breaking core functionality

  1439→5.3.4 For You – OnDevice Personalization (Expanded)
  1440→This is the core of Circle’s privacypreserving recommendation engine.
  1441→Model Architecture
  1442→A matrix factorization model is trained entirely on the device. Each user has a user vector (size 64). Each item (event, Circle, channel, post) has a fixed item vector (computed globally and downloaded once). Recommendations are ranked by cosine similarity between the user vector and item vectors, updated incrementally.
  1443→Lifecycle
  1444→Initialisation: New user starts with a random user vector (or zeros). No coldstart problem because popular items are shown initially.
  1445→Interaction tracking: Each time the user likes, shares, watches, RSVPs, or joins, an interaction event is stored locally (max 10,000 events).
  1446→Training: When the device is idle and charging, a background process runs a single epoch of stochastic gradient descent on the last 1000 interactions.
  1447→Inference: When requesting “For You”, the app computes similarity between current user vector and all candidate items (cached from server, filtered by city, recency, etc.). Top 20 returned.
  1448→Code Example
  1449→dart
  1450→// lib/recommendation/engine.dart
  1451→import 'dart:math';
  1452→import 'package:vector_math/vector_math.dart' as vm;
  1453→class RecommendationEngine {
  1454→  static const int embeddingDim = 64;
  1455→  static const double learningRate = 0.01;
  1456→  late vm.Vector2 _userVector; // Actually a list of doubles, but we use Vector for convenience
  1457→  List<ItemEmbedding> _itemEmbeddings = [];
  1458→  Future<void> loadUserVector() async {
  1459→    final stored = await storage.read('user_vector');
  1460→    if (stored != null) {
  1461→      _userVector = vm.Vector64.fromList(stored.cast<double>());
  1462→    } else {
  1463→      _userVector = vm.Vector64.random(Random(), min: 0, max: 0.01);
  1464→    }
  1465→  }
  1466→  void updateUserVector(List<Interaction> interactions) {
  1467→    for (final interaction in interactions) {
  1468→      final item = _itemEmbeddings.firstWhere((i) => i.id == interaction.itemId);
  1469→      // Simple gradient ascent for positive interactions
  1470→      final similarity = dot(_userVector, item.vector);
  1471→      final gradient = item.vector * (1 - similarity);
  1472→      _userVector += gradient * learningRate;
  1473→    }
  1474→    // Normalise to unit length
  1475→    _userVector = _userVector.normalized();
  1476→    _saveUserVector();
  1477→  }
  1478→  List<Item> getRecommendations(List<Item> candidates, int limit) {
  1479→    final scored = candidates.map((item) {
  1480→      final itemVec = _itemEmbeddings.firstWhere((i) => i.id == item.id).vector;
  1481→      final score = dot(_userVector, itemVec);
  1482→      return MapEntry(item, score);
  1483→    }).toList();
  1484→    scored.sort((a, b) => b.value.compareTo(a.value));
  1485→    return scored.take(limit).map((e) => e.key).toList();
  1486→  }
  1487→  double dot(vm.Vector64 a, vm.Vector64 b) {
  1488→    double sum = 0;
  1489→    for (int i = 0; i < embeddingDim; i++) sum += a[i] * b[i];
  1490→    return sum;
  1491→  }
  1492→}
